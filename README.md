# 红外通信系统 - 上位机与下位机模块

## 项目简介

本项目是一套基于红外无线链路的机械臂远程控制系统，由上位机（STM32F407）和下位机（STM32F103）两个核心模块组成。上位机负责电机驱动与运动控制，下位机作为红外中继节点完成 CAN↔红外 双向协议转换。两个模块通过 38kHz 红外载波进行无线通信，实现机械臂的远程控制和状态监控。

## 设计思想

### 1. 分层解耦 — 硬件无关的协议栈

系统严格遵循"协议层与硬件层分离"的原则。上位机的 `infrared_host` 模块不操作任何红外硬件，仅通过 CAN 总线与下位机交互；下位机才负责 CAN↔红外 的物理层转换。这使得上位机的红外通信逻辑可以完全脱离红外硬件进行调试，也意味着未来替换红外模块为其他无线介质（如蓝牙、Sub-1GHz）时，上位机代码零修改。

### 2. 可靠传输 — 多级确认与重传

红外信道是半双工、易受干扰的无线介质。系统在多个层面保障数据可靠性：

- **CAN 层 ACK**：下位机收到 CAN DATA 帧后立即回 CAN ACK，上位机据此判断下位机是否可达
- **红外层 ACK/NACK**：下位机收到红外数据帧后回红外 ACK（CRC 通过）或 NACK（CRC 失败）
- **RELIABLE\_DATA 帧**：下位机收到此类帧后不立即回 CAN ACK，而是等红外投递成功后才回，实现端到端确认
- **超时重传**：上位机 `IR_Send` 阻塞等待 ACK，超时自动重试（默认 3 次）

### 3. 多模块冗余 — 投票机制

多个下位机模块同时接收同一红外信号时，系统提供两种数据融合策略：

- **`IR_ReadVoted`**：多数派投票，取最多模块一致的数据作为可信结果，适合需要高可靠性的场景
- **`IR_ReadAll`**：去重聚合，返回所有不同的有效数据及其同意模块数，适合需要感知多源信号的场景
- **`IR_Read`**：读取指定模块原始数据，保留方向信息，适合需要判断信号来源的场景

投票窗口机制：首个模块收到新数据后开启 50ms 窗口，等待所有在线模块上报后执行投票；若窗口超时则用已收到的数据投票。

### 4. 自发自收过滤

下位机的红外收发共用空间信道，发送时自身接收器也会捕获信号。系统通过"发送冷却期"机制过滤自发自收：数据帧冷却 80ms，ACK 帧冷却 40ms（ACK 帧更短，因为只有 2 字节且需要尽快接收对方确认）。

### 5. 错峰调度

多模块同时响应会导致 CAN 总线冲突。下位机按模块 ID 低位计算错峰延迟（`module_id & 0x0F) × 3ms`），确保各模块依次响应，避免总线仲裁失败。

## 关键技术

<br />

### 2. 红外物理层协议

基于 NEC 协议变体，采用 38kHz PWM 载波，时序压缩至标准 NEC 的一半：

| 信号   | 高电平(载波) | 低电平(空闲) | 总时长    |
| ---- | ------- | ------- | ------ |
| 起始信号 | 4.5ms   | 2.25ms  | 6.75ms |
| 位 0  | 280μs   | 280μs   | 560μs  |
| 位 1  | 280μs   | 840μs   | 1120μs |
| 停止位  | 280μs   | —       | —      |

一帧数据 = 8 字节数据 + 1 字节 CRC8 = 72 位，最短约 47ms，最长约 87ms。

发送使用 TIM1 产生 38kHz PWM 载波 + TIM3 精确控制位时序（状态机驱动）；接收使用 TIM2 输入捕获，仅捕获上升沿间隔来判断位 0/1。

### 3. CRC8 校验

采用 CRC-8/CCITT 多项式（0x07），初始值 0xFF。每帧 9 字节中前 8 字节为数据，第 9 字节为 CRC，接收端校验失败回 NACK 触发重传。

### 4. 上位机多模块管理

- **链表结构**：`IR_Module_List_t` 以单链表管理最多 8 个下位机模块，支持动态增删
- **数据缓存**：每个模块节点维护 `IR_Module_DataCache_t`，记录原始数据、更新时间戳、CRC 错误计数
- **在线状态机**：`Discovery → Running` 两阶段，发现阶段 Ping 探测，运行阶段轮询保活
- **线程安全**：模块链表操作通过 FreeRTOS 互斥量保护，CAN 接收通过消息队列（深度 20）从 ISR 解耦到任务上下文

### 5. 下位机双队列转发

下位机采用两个独立的环形 FIFO 队列解耦收发：

- **CAN→IR 队列**：CAN 收到 DATA 帧后入队，红外空闲时出队发送
- **IR→CAN 队列**：红外收到数据后入队，CAN 邮箱可用时出队发送

队列深度均为 4，配合错峰调度，有效防止多模块并发导致的丢帧。

### 6. 可靠发送状态机（下位机）

针对 RELIABLE\_DATA 帧实现非阻塞端到端确认：

```
IDLE → SENDING(发红外) → WAIT_ACK(等红外ACK)
  ↑                         │
  │    成功 → 回CAN ACK     │  超时/NACK → 重试
  │                         ↓
  └──── RETRY_WAIT(10ms) ←─┘
         重试耗尽 → 回CAN NACK
```

## 通信协议

### 帧类型定义

| 帧类型             | CAN ID格式         | 方向      | 说明                           |
| --------------- | ---------------- | ------- | ---------------------------- |
| CMD帧            | 0x1\[module\_id] | 上位机→下位机 | 命令帧（PING/RESET/SET\_ID等）     |
| DATA帧           | 0x2\[module\_id] | 上位机→下位机 | 数据帧，下位机立即回CAN ACK并转发红外       |
| ACK帧            | 0x3\[module\_id] | 下位机→上位机 | 确认帧（0xA5A5=ACK, 0x5A5A=NACK） |
| RELIABLE\_DATA帧 | 0x4\[module\_id] | 上位机→下位机 | 可靠数据帧，等红外投递成功后才回CAN ACK      |

### 红外帧格式

```
[8字节数据][1字节CRC8] = 9字节(72位)
```

### CMD 命令码

| 命令           | 值    | 说明                     |
| ------------ | ---- | ---------------------- |
| PING         | 0x01 | 心跳探测                   |
| SEND\_DATA   | 0x02 | 发送数据                   |
| READ\_STATUS | 0x03 | 读取状态                   |
| RESET        | 0x04 | 复位从机                   |
| READ\_SENSOR | 0x05 | 读取传感器                  |
| SET\_ID      | -    | 远程配置模块ID（CMD帧data\[2]） |

## 模块说明

### 1. Arm\_infrared-host\_Lower（上位机）

- **主控芯片**：STM32F407VETx
- **CAN 配置**：双 CAN（CAN1: 电机控制，CAN2: 红外通信 + RobStride）
- **RTOS**：FreeRTOS，任务优先级 Motor\_Drive(4) > IR\_Host\_Task(3)
- **电机支持**：RobStride 01/03（扩展帧）、RM3508、M2006
- **控制算法**：位置环 + 速度环 PID 双闭环，关节级 `Joint_t` 抽象
- **红外协议**：`infrared_host` 模块，链表管理多模块，投票/重传/在线检测
- **PC 通信**：USB-CDC 虚拟串口，收发 6 关节 `Arm_t` 结构体

### 2. U8T6--2（下位机）

- **主控芯片**：STM32F103C8T6
- **红外收发**：TIM1 PWM 载波 + TIM3 发送时序 + TIM2 输入捕获接收
- **协议转换**：CAN↔红外 双向 FIFO 队列转发
- **可靠性**：CRC8 校验、ACK/NACK 确认、RELIABLE\_DATA 端到端确认状态机
- **自发自收过滤**：发送冷却期机制（数据帧 80ms / ACK 帧 40ms）
- **错峰调度**：`(module_id & 0x0F) × 3ms` 延迟响应
- **裸机架构**：主循环轮询，无 RTOS

## 使用说明

### 1. 编译各模块

- 使用 Keil MDK-ARM 分别打开两个模块的项目
- 编译生成 hex 文件

### 2. 烧录程序

- 将上位机程序烧录到 STM32F407
- 将下位机程序烧录到 STM32F103

### 3. 硬件连接

- 上位机 CAN1 连接电机，CAN2 连接红外下位机
- 下位机 CAN 连接上位机，红外模块连接 TIM1/TIM2 对应引脚
- **务必确保 CAN 总线两端并联 120Ω 终端电阻**

### 4. 系统启动

- 上位机启动后初始化 CAN → 红外模块（发现阶段 Ping 探测）→ 电机 → 创建任务
- 下位机启动后初始化红外 → 进入主循环等待 CAN/红外数据
- 红外模块自动发现在线下位机，无需手动配置
